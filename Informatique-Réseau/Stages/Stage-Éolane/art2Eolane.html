STAGE - Entreprise Éolane

Période de formation en entreprise du 20 Mai au 28 Juin 2024 (6 semaines).

Dans cet article je vais présenter l'entreprise où j'ai réalisé mon stage ainsi que les tâches que j'ai pu réaliser au sein du service informatique.

Présentation de Éolane Angers

Éolane est une entreprise française créée en 1975 à Combrée et son cœur de métier est la fabrication et l’assemblage de cartes électronique. Les clients de Éolane proviennent de secteurs divers, on retrouve par exemple des compagnies qui font de la sécurité et de la défense, des constructeurs de véhicules (terrestres et ou aérien) mais aussi plus largement des industriels.

Pictogrammes présentant les types de marchés pour lesquel Éolane produit des cartes.

Aujourd’hui, l’entreprise possède plusieurs sites à travers le monde et les 2 principaux sites de production sont installés dans le Maine-et-Loire, je réalise donc mon stage sur le site d’Angers où se trouve le siège social et actuellement environ 200 personnes y travaillent.

Connexion au réseau :

Restauration d’une backup sur un autre support.

Une demande a été réalisée par un employé, son souhait était que nous réalisions une mise à niveau des composants d’un ordinateur avec un SSD et de la RAM supplémentaire.

La problématique que nous avons rencontrée a été la différence de taille des disques, le disque initial faisais 1T et le disque SSD que nous avions faisait lui 240GB, alors évidemment l’entièreté du disque de 1T n'était pas utilisée, mais concrètement il est souvent plus facile de cloner un disque vers un plus grand disque que l’inverse mais cela est possible.

Nous avons tout d'abord procédé de la manière suivante en utilisant la solution de sauvegarde de l'entreprise active backup de synology par un boot PXE et nous avons pu redéployer la backup sur une machine virtuelle puis une physique.

Nous avons rencontré des problèmes pour redimensionner les partitions et les sauvegardes, certaines sauvegardes prenaient l’ensemble du périphérique en compte et ne ciblait pas uniquement les partitions.

J’ai donc utilisé plusieurs logiciels de clonage de disque dont AOMEI Backupper que je connaissais auparavant et Diskgenius que j’ai découvert sur le moment, du côté de AOMEI j’ai utilisé une version d'essai, car c’est un logiciel avec licence payante en principe, or la version gratuite ne permettait pas de redimensionner correctement les partitions et elle ne faisait pas la restauration de toutes les partitions de manière ordonnée. J’évoque la notion d’ordre car il en est question dans l’écriture et la lecture des partitions, on peut parler aussi de séquence ou d’ordre d’amorçage.

Les partitions doivent être positionnées dans un ordre précis similaire celui-ci EFI > MSR > Local disk (C:) > Partition de récupération.

Lors d’un formatage de disque par exemple dans l’explorateur de fichier la partition MSR n’est pas expressément supprimée et elle est utile au processus de démarrage du système et par ailleurs certains logiciels de clonage ne la détecte tout simplement pas ou du moins ce n’était pas le cas pour AOMEI dans sa version gratuite.

J’ai donc déployé une sauvegarde sur une machine virtuelle avec Active backup et je souhaitais connecter en USB mon SSD mais j’ai rencontré une erreur, VirtualBox n’a pas réussi à monter mon disque, finalement nous avons utilisé un pc physique et j’ai pu installer la solution Diskgenius et simplement cloner mon disque avec l’ensemble des partitions.

Création d'un package Chocolatey

J'ai mentionné précédemment un script shell qui utilise des gestionnaires de paquets Windows, comme Chocolatey et Winget, pour installer des applications. Ces outils open source permettent d'automatiser l'installation de logiciels sur les postes de travail.

L'objectif était de répondre à un besoin spécifique : assurer la compatibilité avec les anciennes bases Access de Microsoft. Pour cela, nous avons ajouté le driver nécessaire à la liste des packages installés par le script. Voici comment nous avons procédé :

Étape procédure :

Rejoindre GitLab :

  • GitLab est une plateforme de travail collaboratif, similaire à GitHub, qui facilite le partage de code source.

Installer Git :

  • Git est nécessaire pour utiliser GitLab. Il permet de suivre les modifications d'un projet et d'effectuer des opérations comme les commits et les pull requests.

Installer Chocolatey :

  • Nous avons installé Chocolatey en utilisant des commandes PowerShell spécifiques.

Télécharger le code du projet :

  • Avec la commande git clone, nous avons récupéré le code source du projet depuis GitLab.

Créer le package :

  • Nous avons utilisé Visual Studio Code pour créer les fichiers du nouveau package avec la commande choco new nomdupackage. Ensuite, nous avons modifié certains paramètres dans le fichier .nuspec, comme la version, l'auteur, et la description.

Construire et vérifier le package :

  • Avec la commande choco pack nompackage, nous avons construit le package. Enfin, nous avons vérifié la configuration de la clé API GitLab pour envoyer les modifications (commit).

Ces étapes nous ont permis de créer un package Chocolatey pour automatiser l'installation des drivers nécessaires sur les postes de travail.
